# Domains and availability

## Честное ограничение

Скилл **не выполняет** автоматическую проверку регистрации домена в реальном времени. Фразы вроде «✓ available» без проверки через регистратора или API **недопустимы**.

Допустимо:

- Пометить кандидата как *«нужна проверка у регистратора / WHOIS»*.
- Указать **риск типовых конфликтов** (очень generic слово, известный бренд в смежной категории).
- После **`Task(researcher)`** — ссылка на публичные страницы аукционов/парковки, если исследование нашло явные сигналы.

## TLD — ориентиры

| TLD | Обычно когда |
|-----|----------------|
| .com | Универсально, максимум доверия у широкой аудитории |
| .io | Тех-стартапы, dev-аудитория |
| .dev | Продукты для разработчиков (HTTPS по политике зоны — учитывать при плане) |
| .ai | AI/ML-нарратив (цена/спрос выше) |
| .app | Приложения |
| .co | Альтернатива .com |
| .design, .tech, .eco и др. | Нишевые истории |

## Workflow

1. Сгенерировать **короткий список имён** (5–7), уже прошедших скоринг.
2. Для каждого — **целевые TLD** (например `.com` + `.io`).
3. Для топ-3 — описать **вариации** при занятости: опечатка бренда, добавка слова, другой compound, другой TLD.
4. **Next steps для оператора:** зайти к регистратору, проверить WHOIS, при необходимости trademark search / юрист.

## Если первый выбор занят

- Не зацикливайся на одном корне: часто выигрывает **сдвиг метафоры**, а не дефис/цифра.
- Рассмотри **защитный набор** (одно имя + несколько TLD) только если бюджет и стратегия это оправдывают — явно как рекомендация, не как обязательность.

## Соцсети и единый @handle

Отметь в выводе отдельным подпунктом: *«проверить @handle на ключевых площадках»* — без имитации проверки, если нет доступа к инструментам.
