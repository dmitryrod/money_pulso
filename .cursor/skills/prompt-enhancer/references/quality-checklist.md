# Quality checklist (final success lock)

Перед выдачей улучшенного промпта проверь по пунктам:

- [ ] **Intent** — сохранён исходный смысл запроса пользователя; задача не подменена на другую.
- [ ] **Self-contained** — есть цель, контекст (где уместно), input/output ожидания, constraints.
- [ ] **Cursor scope** — для правок в коде: явные пути/файлы **или** явная инструкция сначала прочитать релевантные файлы.
- [ ] **Done when** — бинарные или проверяемые критерии готовности; не «сделай хорошо».
- [ ] **Stop / gates** — stop condition; «спроси перед…» для deps, удаления файлов, миграций схемы, секретов.
- [ ] **Task** — `Task(subagent_type="...")` только с реальными агентами из [`.cursor/agents/`](../../../agents/); `prompt` и `description` осмысленны.
- [ ] **No framework noise** — в **финальном** пользовательском промпте нет бесполезных имён внешних prompt-frameworks.
- [ ] **Lean context** — нет токсичного «just in case»; каждая секция load-bearing.
- [ ] **Examples** — 0–2 short examples **только** если иначе формат не стабилен; иначе пропустить.
- [ ] **Простая задача** — промпт не раздут до feature-workflow без причины.

Дополнительно при правке **markdown агента/скилла** как артефакта:

- [ ] YAML frontmatter цел; intent не изменён без явного согласия в запросе.
- [ ] Ссылки на `.cursor/` относительные и валидные по плану репо.
