# CRO — гипотеза для страницы «Быстрый старт / Установка»

*Дата: 2026-04-27*

## Гипотеза

Если в первом экране блока установки показать **одну команду** `docker compose up` (или `uv run`) и ссылку на `.env.example`, доля дошедших до «первый скринер создан» вырастет vs длинный prose-only блок.

## Метрика

- Событие: `setup_docs_scroll_to_env` / `compose_copy_click` (если появится аналитика).
- Прокси: время до первого issue «как запустить» в GitHub (качественно).

## Что тестируем

- **A:** текущий порядок секций в `app/docs/README.md`.
- **B:** вынести «Минимальный старт» выше Marp-презентаций для новых пользователей (только если продуктовая команда согласна — сейчас док объединяет много целей).

## Ограничения

Не ломать навигацию для существующих пользователей Marp; A/B только на отдельной landing, не обязательно в монолитном README.

## Статус

`deferred` — требуется отдельная страница или split README по решению владельца.
